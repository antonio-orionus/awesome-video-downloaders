# Contribution Guidelines

Thanks for taking the time to contribute!

## Adding an entry

- Open a pull request that adds the entry to the appropriate section in `readme.md`.
- Format: `- [name](url) - Short description ending with a period.`
- Sort entries by relevance within their section.
- The entry must be a user-facing app for downloading video or audio. Libraries, plugins, and command-line wrappers without a UI belong elsewhere.
- The project must be actively maintained, have a usable README, and not be a duplicate of an existing entry.

## Updating an entry

If a project is renamed, abandoned, or no longer matches the scope, open a PR to update or remove it.

## Removing an entry

Open a PR with a brief reason (archived, broken, out of scope, etc.).

## Style

- Follow the existing list format.
- Use sentence case for descriptions.
- Keep descriptions short — one sentence is best.
- Don't use marketing language.
